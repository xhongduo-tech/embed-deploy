本页帮助你在纯文本检索、更强语义（Qwen3）与图文多模态之间做选择，并给出可直接复制的调用示例。 所有路径均由前置网关（Nginx）转发至对应推理容器。

服务已就绪

不知道怎么选？三套「开箱即用」组合

先按业务选一组即可；同一套 RAG 链路里，Embedding 与 Reranker 建议不要混用不同家族（例如 BGE 向量 + Qwen 重排），除非你做过后端实验验证相关性。

默认推荐 · 稳妥

方案 A：BGE 全家桶

已有系统、要高并发、只要中文/英文文本检索。接口最成熟，旧代码零改动即可对接。

  • 向量化 → /v1/embeddings · bge-m3
  • 重排序 → /v1/rerank · bge-reranker-v2-m3
文本质量优先

方案 B：Qwen3 文本双模型

纯文本场景下希望语义更贴近 Qwen 生态，可与 Qwen 大模型流水线搭配；由 llama.cpp 加载 GGUF。

  • 向量化 → /v1/qwen-embed · qwen3-embedding-8b
  • 重排序 → /v1/qwen-rerank · qwen3-reranker-8b
含图 / 截图

方案 C：Qwen3-VL 多模态

查询或文档侧含图片、扫描件、截图，需要视觉编码时选用；服务端需加载主 GGUF 与 mmproj（部署见项目 README）。

  • 向量化 → /v1/qwen-vl-embed · Qwen3-VL-Embedding-2B
  • 重排序 → /v1/qwen-vl-rerank · Qwen3-VL-Reranker-2B

四步快速决策

仍不确定时，从上到下答四个问题即可落到具体路径。

1
有没有图像输入？

有（商品图、截图、PDF 渲染图等）→ 走 VL 的 /v1/qwen-vl-*；没有 → 继续下一步。

2
要不要兼容历史接口？

老系统已写死 /v1/embeddings、/v1/rerank → 选 BGE；新项目可自由选择。

3
更在意吞吐还是语义？

高 QPS、低延迟、动态批处理 → BGE + Infinity；更重向量语义、与 Qwen 系列统一 → Qwen3 文本。

4
是否要多节点负载？

BGE 的 Embedding 在网关侧为三节点，Rerank 为双节点；Qwen3 文本向量化（Qwen3-Embedding-8B）为单节点，VL 与各 Rerank 仍为单节点。若要强容灾，优先 BGE 路径或在前端自己做重试。

Embedding：该用哪条路径？

把「文本变成向量」；用于向量库检索、相似度计算。

Embedding：路线、场景、路径与模型字段
路线 典型场景 HTTP 路径 model 字段 引擎
BGE 通用文本检索、要高并发、与 LangChain 示例最贴合 /v1/embeddings bge-m3 Infinity · 三节点
Qwen3 纯文本、希望与 Qwen3 系列语义对齐 /v1/qwen-embed qwen3-embedding-8b llama.cpp · 单节点
Qwen3-VL 图文混合、需要对图像编码进同一向量空间 /v1/qwen-vl-embed Qwen3-VL-Embedding-2B llama.cpp · 多模态（+mmproj）

Reranker：该用哪条路径？

对候选文档「精排」；通常接在向量召回之后，缩小 Top-K、提升相关性。

Reranker：路线、场景、路径与模型字段
路线 典型场景 HTTP 路径 model 字段 引擎
BGE 纯文本重排、双节点负载、与旧系统一致 /v1/rerank bge-reranker-v2-m3 Infinity · 双节点
Qwen3 纯文本、与 Qwen3-Embedding 搭配做 RAG /v1/qwen-rerank qwen3-reranker-8b llama.cpp · 单节点
Qwen3-VL 查询或文档侧含图像时的重排 /v1/qwen-vl-rerank Qwen3-VL-Reranker-2B llama.cpp · 多模态（+mmproj）

双引擎说明（知道即可）

底层两套推理栈并行存在，通过路径区分；对你而言选对路径与 model 名称即可。

Infinity

服务 BGE 系列（HF 格式），动态批处理、偏吞吐；暴露标准 /v1/embeddings、/v1/rerank。

llama.cpp

服务 Qwen3 全系 GGUF；VL 模型需额外加载 mmproj 才能启用视觉分支。详见仓库 README.md 与 docker-compose.yml。

1. BGE-M3 · 文本向量化（三节点，openai 包）

import OpenAI from "openai";

const openai = new OpenAI({
  baseURL: "http://api-embed.cs.icbc/v1",
  apiKey: "sk-internal",
});

const r = await openai.embeddings.create({
  model: "bge-m3",
  input: "测试 Node.js 接入",
});

const BASE = "http://api-embed.cs.icbc";

from langchain_openai import OpenAIEmbeddings

)

query_vector = embeddings.embed_query("内网部署方案")
docs_vectors = embeddings.embed_documents([
    "文件检索工具初始化中...",
    "采用 GPU 算力底座。",
])
